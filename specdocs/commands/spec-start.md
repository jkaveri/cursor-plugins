# Spec start

Use the **write-feature-spec** skill.

1. Ask for the **feature name** (kebab-case slug) or propose one from context.
2. Ask for **problem, users, and success criteria** if not already provided.
3. Create or update **`docs/specs/<feature>/`** starting with **`spec.md`**, then add **`plan.md`**, **`tasks.md`**, **`rollout.md`**, **`test-cases.md`** only where they add clear value—no empty boilerplate unless I ask for stubs.
4. If **`docs/README.md`** or **`docs/planning/features.md`** should link to this feature, suggest a single line addition—do not rewrite the whole index unless I ask.
